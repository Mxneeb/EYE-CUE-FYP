# EYE-CUE (FYP) — Real-time Navigation Assist

EYE-CUE is a real-time navigation assistance prototype for visually impaired users. It fuses **monocular depth estimation** (Depth Anything V2) with **semantic segmentation** (TopFormer on ADE20K) to detect obstacles and produce **spoken movement guidance**, plus optional safety and utility features (SOS, time/weather, clips).

## Features (and why they’re useful)

- **Dual-stream perception (Depth + Segmentation)**: more robust obstacle understanding than either stream alone (depth gives distance; segmentation gives “what it is”).
- **Obstacle fusion + risk reasoning**: filters walkable classes (floor/road/sidewalk etc.) and focuses guidance on hazards that matter.
- **Fuzzy-logic path planning**: produces stable, human-friendly instructions (less jitter than hard thresholds; better handling of “almost blocked” cases).
- **Voice navigation guidance (TTS)**: hands-free feedback so the user can keep moving without looking at a screen.
- **Gemma multimodal assistant (on-device)**:
  - **Surroundings description**: quick environmental summary tailored for blind/low-vision users.
  - **Wardrobe suggestion**: identifies clothing items/colors and recommends matching combinations.
  - **Pro**: heavy work runs in background threads so the live navigation loop stays responsive.
- **Time + weather announce** (Open‑Meteo): fast utility info without needing an API key.
- **Distance sonar mode**: non-verbal beeps that scale with hazard level (useful in noisy environments or when you don’t want speech).
- **Clip buffer (“save last 30 seconds”)**: captures what happened right before an incident without recording everything.
- **SOS alert module**: can send an emergency alert with photo + location (requires configuration).

## Project structure

- `main.py`: entrypoint used by `run_navigation.bat` (launches `nav_assist.app`).
- `nav_assist/`: core pipeline (workers, fusion, planner, audio, visualization, Gemma assistant).
- `obs-tackle/third_party/Depth-Anything-V2/`: Depth Anything V2 vendor code.
- `convert_topformer_onnx.py`: convert/export TopFormer to `topformer.onnx` (one-time).
- `sos/`: SOS setup + database + Telegram/location sender.
- `tests/`: planner and obstacle logic tests.
- Output folders (ignored by git): `screenshots/`, `video_clips/`, `gemma_snapshots/`, `recordings/`, `bookmarks/`.

## Requirements

- **Windows 10/11**
- **Python 3.10+** (3.9+ often works, but 3.10+ is recommended)
- A **webcam**
- Optional: NVIDIA GPU (CUDA) for better real-time performance

### Model assets (not committed)

This repo expects local model files (ignored by `.gitignore`):

- **Depth Anything V2 checkpoint**: `depth_anything_vits14.pth` (see `nav_assist/config.py`)
- **TopFormer ONNX**: `topformer.onnx` (generated by `convert_topformer_onnx.py`)
- **Optional Gemma assets** (for the AI description/wardrobe keys):
  - `gemma-4/gemma-4-E2B-it-Q3_K_M.gguf`
  - `gemma-4/mmproj-F16.gguf`
- **Optional Piper voice** (for neural TTS): `piper_voices/en_US-lessac-high.onnx` (+ its `.json`)

If your checkpoint filename differs, either rename it to match `nav_assist/config.py` or update `DEPTH_CKPT` there.

## Setup & Run (Windows)

### Option A (recommended): batch scripts

1) Create the virtual environment and install dependencies:

```bat
setup_venv.bat
```

2) Run the app:

```bat
run_navigation.bat
```

### Option B: manual

```bat
python -m venv venv
venv\Scripts\activate.bat
pip install --upgrade pip
pip install -r requirements.txt
python convert_topformer_onnx.py
python main.py
```

## Controls (in-app)

- `Q` / `Esc`: quit
- `S`: save screenshot
- `M`: mute/unmute navigation speech
- `D`: AI surroundings description (Gemma)
- `W`: AI wardrobe suggestion (Gemma)
- `T`: announce time + weather
- `C`: announce time/date (offline)
- `A`: toggle sonar beeps
- `V`: save last ~30 seconds as a clip
- `E`: trigger SOS

## SOS setup (high level)

The SOS module needs a configured contact + Telegram bot token and may use network-based location. Start here:

- `sos/setup.py`

Then test:

- `python -m sos.sos`

## Notes

- Large assets (models, screenshots, clips, etc.) are intentionally ignored by `.gitignore`.
- If you’re missing dependencies for audio playback (e.g. `aplay`, Piper), the app will degrade gracefully (speech/sonar may be silent depending on what’s installed).
